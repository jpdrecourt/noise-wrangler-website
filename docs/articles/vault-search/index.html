Vault Search

Hybrid semantic and keyword search over markdown vaults, running locally.

Get it

  1. Clone from github.com/jpdrecourt/vault-search
  2. Install dependencies: pip install -r requirements.txt
  3. Edit config.json to point at your notes directory, then index: python indexer.py
  4. Search: python search.py "your query" --top_k 10

Python 3.11+. GPU optional (falls back to CPU). MIT licensed.

The problem

I have around 600 notes in an Obsidian vault. Full-text search finds exact words. It misses concepts. Searching for "feedback in performance" won't surface a note about audience response shaping improvisation decisions, even though that's exactly what I'm looking for.

Two searches, one score

The tool runs two searches in parallel. FAISS handles semantic similarity: it embeds your query and every chunk of text into 768-dimensional vectors, then finds the closest matches by meaning. BM25 handles keyword matching: the classic information-retrieval algorithm that ranks documents by term frequency.

Both scores get normalised and fused into a single ranking. The balance is adjustable (the --alpha flag), defaulting to 60% semantic, 40% keyword. This catches both conceptual neighbours and exact term matches that pure vector search misses.

Incremental indexing

The indexer tracks file checksums. On a typical run it only re-embeds new or modified files, finishing in a few seconds. A full rebuild of ~600 notes takes about 30 seconds on GPU.

Plugging it into Claude Code

I use this through a Claude Code skill: a small markdown file that tells Claude how to call the search script. When I ask about a topic that might be in my notes, Claude runs the search, reads the top results, and brings that context into the conversation. No server process, no protocol layer. The repo also includes an MCP server for setups that prefer that route.
